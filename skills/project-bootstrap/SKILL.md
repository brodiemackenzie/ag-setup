# Project Bootstrap Skill

This skill contains the automated playbooks and tools required to initialize a new, version-controlled software development project cleanly scaffolded with specialized process frameworks (e.g., Spec-Driven Development).

---

## Overview

The bootstrap command is a global utility that automates greenfield project directory provisioning, removes boilerplate code or framework git tracking artifacts, binds standard origin remotes, and copies version-controlled process templates safely without copying internal Git metadata.

---

## Playbook

### Initializing a New Project

To initialize a new repository using the global installer:
1. Ensure your global template folder is symlinked cleanly:
   ```bash
   ln -s ~/projects/ag-setup/templates/ ~/.gemini/templates
   ```
2. Run the bootstrapper command:
   ```bash
   bootstrap.sh <project_name> <github_repo_url> [--process <process_slug>] [--scaffold <framework>]
   ```
   * `<project_name>`: The directory name where the project will be created under `~/projects/`.
   * `<github_repo_url>`: Target GitHub remote URL.
   * `--process`: Target agent workflow template to install (default: `sdd-process`).
   * `--scaffold`: Optional framework boilerplate scaffold (e.g., `nextjs`, `python`, or `none`).

### 🎙️ Interactive Conversational Workflow (Alternative to CLI)

If the user types `/bootstrap`, `bootstrap`, or `start a new project` inside their active Antigravity chat window, you must dynamically take the lead as the **Conversational Installer Agent**:

1. **Parameters Collection**: Ask the user for the required parameters **one at a time** in chat (do not dump them all at once):
   * *Step 1*: Ask for the **Project Name** (e.g., *"Great! Let's initialize a new workspace. What would you like to name the project folder?"*).
   * *Step 2*: Ask for the **Target GitHub SSH URL** (e.g., *"Got it. What is the target GitHub remote repository SSH URL to bind to this project?"*).
   * *Step 3*: Ask for the **Scaffolding Choice** (e.g., *"Should we scaffold a basic Python structure, a Next.js app, or start with a blank slate? Please reply with 'python', 'nextjs', or 'none'."*).
2. **Formulate & Run the Shell Command**: Once all parameters are collected, formulate the appropriate CLI execution string and run it on behalf of the user using the `run_command` tool:
   ```bash
   # Execute the global bootstrapper CLI:
   ~/skills/project-bootstrap/scripts/bootstrap.sh <project_name> <github_url> --scaffold <framework>
   ```
3. **Present the Outcome**: Display the successful provisioning console logs to the user, confirm that all custom rules and skills are copied, and guide the user to open the new directory to start their proposal interview.

### Execution Mechanics

The script executes the following steps sequentially:
1. **Directory Verification**: Ensures that `~/projects/<project_name>` does not already exist to prevent data loss.
2. **Workspace Creation & Git Init**: Creates the directory and initializes git.
3. **Scaffolding Boilerplate**: Runs standard generators if `--scaffold` is specified.
4. **Clean Boilerplate metadata**: Scans recursively to delete any nested `.git` folders generated by boilerplate packages.
5. **Remote Bind**: Binds the origin repository.
6. **Safe Template RSync**: Copies template configurations cleanly without duplicating template git histories:
   ```bash
   rsync -av --exclude='.git' "~/.gemini/templates/<process_slug>/_agents/" "~/projects/<project_name>/.agents/"
   ```
7. **SDD Interview Kickoff**: Triggers the initial speculative interview to draft the project proposal.
